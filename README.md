# ChatGPT-RTL

**כלי קהילתי (לא רשמי) שמוסיף יישור מימין לשמאל (RTL) לאפליקציית הדֶסקטופ של ChatGPT** — האפליקציה המאוחדת שמשלבת את מצב "ChatGPT Work" ואת מצב "Codex".

עברית וערבית מתיישרות לימין, אנגלית נשארת מיושרת לשמאל, ובלוקים של קוד ונוסחאות מתמטיות נשארים LTR וקריאים כמו שצריך.

הכלי בונה אפליקציה **נפרדת** בשם "ChatGPT-RTL". האפליקציה המקורית לעולם לא משתנה וממשיכה להתעדכן כרגיל, ומנגנון עדכון אוטומטי שרץ ברקע מחיל מחדש את הפאץ' באופן אוטומטי בכל פעם שהאפליקציה המקורית מתעדכנת.

> **In one line:** a one-command tool that builds a separate, RTL-aware copy of the ChatGPT/Codex desktop app (macOS & Windows), and keeps it patched across updates.

**מאגר הקוד:** https://github.com/Asher-pro/ChatGPT-app-RTL

## דרישות מקדימות

- **macOS** (Apple Silicon או Intel), או **Windows 10/11**.
- **Node.js** מותקן (מספק את `node` ואת `npx`). ניתן להוריד מ-<https://nodejs.org>.
- גישה חד-פעמית לרשת (להורדת כלי האריזה `@electron/asar`).
- אפליקציית הדסקטופ הרשמית של ChatGPT כבר מותקנת (ב-macOS: `/Applications/ChatGPT.app`).

## התקנה — שורה אחת

זו הדרך המומלצת והפשוטה ביותר. מעתיקים, מדביקים בטרמינל, וזהו.

### macOS

```bash
curl -fsSL https://raw.githubusercontent.com/Asher-pro/ChatGPT-app-RTL/main/chatgpt-rtl.sh | bash
```

השורה הזו בונה את `~/Applications/ChatGPT-RTL.app`, מתקינה את מנגנון העדכון האוטומטי שרץ ברקע, ופותחת את האפליקציה.

### Windows (PowerShell)

```powershell
irm https://raw.githubusercontent.com/Asher-pro/ChatGPT-app-RTL/main/chatgpt-rtl.ps1 | iex
```

השורה הזו בונה עותק תחת `%LOCALAPPDATA%\ChatGPT-RTL`, יוצרת קיצור דרך בשם "ChatGPT RTL" בשולחן העבודה, ומתקינה משימה מתוזמנת (Scheduled Task) לעדכונים אוטומטיים.

> **סטטוס בדיקות**
> - **macOS** — נבדק מקצה לקצה ועובד.
> - **Windows** — נכתב במקביל לגרסת ה-macOS, אך **טרם נבדק** על מכונת Windows אמיתית. ודאו שהוא פועל כראוי אצלכם לפני שאתם מסתמכים עליו.

### הערות חשובות ל-macOS

- **סגרו קודם את ChatGPT המקורי** אם הוא פתוח. שתי האפליקציות חולקות זהות והתחברות (ההתחברות הקיימת שלכם עוברת אוטומטית לעותק החדש), ולכן יש להריץ אחת בכל פעם.
- בהפעלה הראשונה, macOS יבקש אישור למחזיק המפתחות ("Safe Storage") — זה חד-פעמי.
- אם החלון נפתח ריק בפעם הראשונה — סִגרו ופִתחו שוב.

## הרצה עם דגלים (flags) דרך שורה אחת

לא קל להעביר דגלים דרך pipe, ולכן כדי להשתמש באפשרויות יש שתי דרכים: או להוריד קודם את הסקריפט ואז להריץ אותו, או (ב-macOS) להעביר ארגומנטים אחרי `bash -s --`.

### macOS — דגל דרך pipe

```bash
curl -fsSL https://raw.githubusercontent.com/Asher-pro/ChatGPT-app-RTL/main/chatgpt-rtl.sh | bash -s -- --no-watch
```

### macOS — הורדה ואז הרצה (עובד עם כל הדגלים)

```bash
curl -fsSLO https://raw.githubusercontent.com/Asher-pro/ChatGPT-app-RTL/main/chatgpt-rtl.sh
bash chatgpt-rtl.sh --restore
```

### Windows — הורדה ואז הרצה (עבור דגלים)

```powershell
irm https://raw.githubusercontent.com/Asher-pro/ChatGPT-app-RTL/main/chatgpt-rtl.ps1 -OutFile chatgpt-rtl.ps1
powershell -ExecutionPolicy Bypass -File chatgpt-rtl.ps1 -Restore
```

## אפשרויות (דגלים)

| דגל (macOS / Windows) | מה הוא עושה |
|---|---|
| *(ללא דגל)* | בנייה + מנגנון עדכון אוטומטי ברקע + פתיחת האפליקציה |

| `--auto` / `-Auto` | *(פנימי)* בנייה מחדש רק אם גרסת האפליקציה השתנתה — משמש את מנגנון העדכון |

## איך זה עובד

האפליקציה מבוססת **Electron**, וטוענת את הממשק שלה **מרחוק** מ-`chatgpt.com` (בשני המצבים — Work ו-Codex). לכן ההזרקה מתבצעת מתהליך ה-**main** ולא מקבצים מקומיים:

- הוק (hook) על כל `web-contents-created` מזריק CSS באמצעות `insertCSS` — פעולה זו רצה בשכבת Electron ולכן **עוקפת את ה-CSP** של הדף — יחד עם סקריפט bidi קטן.
- ה-CSS מחיל `unicode-bidi: plaintext` על פסקאות, כותרות ורשימות (זיהוי כיוון לכל שורה בנפרד), ומבודד `pre` / `code` / KaTeX / Monaco כ-LTR.
- ה-JS קובע `dir="auto"` לפי התוכן, בעזרת `MutationObserver` **קליל** שסורק רק צמתים חדשים שנוספו, ובזמן idle בלבד (ללא האזנה לכל תו — כדי לא להעמיס על ה-renderer בזמן שתשובה זורמת).

### שלבי הבנייה ב-macOS

1. העתקת האפליקציה אל `~/Applications/ChatGPT-RTL.app` (באפליקציה המקורית לא נוגעים).
2. חילוץ `app.asar`, שיכוב המודולים הנייטיביים (`app.asar.unpacked`), הזרקת ה-loader לנקודת הכניסה הראשית, ואריזה מחדש **תוך שמירה על כל מודול נייטיבי**.
3. חישוב מחדש של ה-hash של `ElectronAsarIntegrity` ועדכון `Info.plist` — כך שבדיקת התקינות עוברת מבלי לגעת ב-fuse.
4. חתימה מחדש (ad-hoc) **מבפנים החוצה** של כל framework / helper / library, עם הרשאת `disable-library-validation` (חובה — אחרת macOS קורס בבדיקת Library Validation).

### המקבילה ב-Windows

העתקה לתיקייה עם הרשאות כתיבה, אותה הזרקה, והחלפת ה-hash בתוך קובץ ה-`.exe` (או השבתת ה-integrity fuse כפתרון חלופי).

## עדכונים

עדכון של האפליקציה מוחק כל פאץ' שהוחל, ולכן הכלי מתקין מנגנון שמזהה את העדכון ובונה מחדש באופן אוטומטי:

- **macOS** — `LaunchAgent` (בנתיב `~/Library/LaunchAgents/com.chatgpt-rtl.updater.plist`) עוקב אחרי ה-`Info.plist` של המקורי; כשהגרסה משתנה, הוא בונה מחדש את עותק ה-RTL.
- **Windows** — משימה מתוזמנת ("ChatGPT RTL Updater") רצה בכניסה למערכת (logon) וגם מדי יום, ובונה מחדש אם הגרסה השתנתה.

## פתרון תקלות

- **"האפליקציה פגומה / לא ניתן לפתוח אותה"** — הריצו מחדש את המתקין; החתימה מתבצעת בסוף התהליך. אם צריך:
  ```bash
  xattr -dr com.apple.quarantine ~/Applications/ChatGPT-RTL.app
  ```
- **קריסה מיידית בהפעלה** — כמעט תמיד מדובר בבעיית חתימה; הרצה חוזרת בונה הכול נקי מאפס.
- **הטקסט לא מיושר** — ודאו שאתם מסתכלים על עותק ה-RTL ולא על המקורי. ה-RTL מופיע רק על טקסט שמכיל בפועל אותיות בעברית או בערבית.
- **חזרה ל-LTR אחרי עדכון** — מנגנון המעקב אמור לתקן זאת; לתיקון מיידי, פשוט הריצו מחדש את המתקין.
