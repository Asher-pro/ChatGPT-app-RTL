# ChatGPT‑RTL

יישור **מימין לשמאל (RTL)** לאפליקציית ChatGPT לדסקטופ — זו שמאחדת את מצב **ChatGPT Work** ומצב **Codex**. עברית וערבית מיושרות ימין, אנגלית נשארת שמאל, ובלוקי קוד/נוסחאות נשארים LTR וקריאים.

הכלי בונה **אפליקציה נפרדת** (`ChatGPT‑RTL`) — האפליקציה המקורית לא נגעת וממשיכה להתעדכן כרגיל. תיקון שמתקין את עצמו מחדש **אוטומטית** בכל פעם שהאפליקציה המקורית מתעדכנת.

> **English, in one line:** a one‑command tool that builds a separate, RTL‑aware copy of the ChatGPT/Codex desktop app (macOS & Windows), and keeps it patched across updates.

## דרישות מוקדמות

- **macOS** (Apple Silicon / Intel) או **Windows 10/11**.
- **Node.js** מותקן (מספק `node` ו‑`npx`). התקנה: <https://nodejs.org>.

## התקנה — פקודה אחת

### macOS ✅ (מאומת מקצה לקצה)

```bash
bash chatgpt-rtl.sh
```

בונה את `~/Applications/ChatGPT‑RTL.app`, מתקין עדכון אוטומטי ברקע, ופותח את האפליקציה.

> **סגור קודם את ChatGPT המקורי** אם הוא פתוח. שתי האפליקציות חולקות זהות והתחברות — ההתחברות שלך כבר עוברת, ומריצים אחת בכל פעם.
>
> בהרצה הראשונה macOS יבקש אישור keychain ("Safe Storage") — חד‑פעמי. אם החלון נפתח ריק פעם אחת, סגור ופתח שוב.

### Windows ⚠️ (מקבילה מלאה; יש לאמת על מכונת Windows)

ב‑PowerShell (Windows PowerShell 5.1 מומלץ):

```powershell
powershell -ExecutionPolicy Bypass -File chatgpt-rtl.ps1
```

בונה עותק ב‑`%LOCALAPPDATA%\ChatGPT‑RTL`, יוצר קיצור דרך "ChatGPT RTL" בשולחן העבודה, ומתקין Scheduled Task לעדכון אוטומטי.

## אפשרויות

| דגל (macOS / Windows) | פעולה |
|---|---|
| *(ללא)* | בנייה + עדכון אוטומטי ברקע + פתיחה |
| `--no-watch` / `-NoWatch` | בנייה בלבד, בלי עדכון אוטומטי |
| `--restore` / `-Restore` | הסרה מלאה — מוחק את עותק ה‑RTL ואת מנגנון העדכון (המקורית לא נגעת) |
| `--auto` / `-Auto` | *(פנימי)* בונה מחדש רק אם גרסת האפליקציה השתנתה — מופעל ע"י ה‑watcher |

## איך זה עובד

האפליקציה היא **Electron** שטוענת את ה‑UI **מרחוק** מ‑`chatgpt.com` (גם ל‑Work וגם ל‑Codex). לכן ההזרקה נעשית מ**תהליך ה‑main** ולא מקבצים מקומיים:

- הוק על כל `web-contents-created` מזריק CSS דרך `insertCSS` — פועל ברמת Electron ולכן **עוקף את ה‑CSP** של הדף — ותסריט bidi קטן.
- ה‑CSS מפעיל `unicode-bidi: plaintext` על פסקאות/כותרות/רשימות (זיהוי כיוון לכל שורה), ומבודד `pre/code/KaTeX/Monaco` כ‑LTR.
- ה‑JS מסמן `dir="auto"` לפי תוכן, עם `MutationObserver` **קליל** שסורק רק צמתים חדשים בזמן idle (בלי האזנה לכל תו — כדי לא להעמיס בזמן סטרימינג).

תהליך הבנייה ב‑macOS:

1. העתקת האפליקציה ל‑`~/Applications/ChatGPT‑RTL.app` (המקורית לא נגעת).
2. חילוץ `app.asar`, שכבת‑על של המודולים הנייטיביים (`app.asar.unpacked`), הזרקת ה‑loader ל‑main entry, ואריזה מחדש **תוך שמירה על כל מודול נייטיבי**.
3. חישוב מחדש של hash ה‑`ElectronAsarIntegrity` ועדכונו ב‑`Info.plist` — כך בדיקת התקינות עוברת בלי לגעת ב‑fuse.
4. חתימה מחדש ad‑hoc **מבפנים החוצה** של כל framework/helper/ספרייה, עם entitlement של `disable-library-validation` (הכרחי — אחרת macOS קורס ב‑Library Validation).

ב‑Windows המקבילה: העתקה לתיקייה כתיבה, אותה הזרקה, והחלפת ה‑hash בתוך ה‑`.exe` (או כיבוי ה‑integrity fuse כ‑fallback).

## עדכונים

עדכון של האפליקציה המקורית **מוחק כל תיקון**, ולכן הכלי מתקין מנגנון שמזהה את זה ובונה מחדש אוטומטית:

- **macOS** — `LaunchAgent` (`~/Library/LaunchAgents/com.chatgpt-rtl.updater.plist`) שמנטר את `Info.plist` של המקורית; כשהגרסה משתנה — בונה מחדש את עותק ה‑RTL.
- **Windows** — Scheduled Task ("ChatGPT RTL Updater") שרץ ב‑logon ומדי יום, ובונה מחדש אם הגרסה השתנתה.

## אימות (macOS)

```bash
# חתימה תקינה
codesign --verify --deep --strict ~/Applications/ChatGPT-RTL.app && echo OK

# מנגנון העדכון טעון
launchctl list | grep chatgpt-rtl
```

ואז פתח את האפליקציה וכתוב עברית בתיבת ההודעה (ב‑Work וב‑Codex) — הטקסט אמור להתיישר ימין, אנגלית שמאל, ובלוק קוד להישאר LTR.

## הסרה

```bash
bash chatgpt-rtl.sh --restore                # macOS
powershell -File chatgpt-rtl.ps1 -Restore    # Windows
```

מוחק את עותק ה‑RTL ואת מנגנון העדכון. האפליקציה המקורית לא נגעה מלכתחילה.

## פתרון תקלות

- **"האפליקציה פגומה / לא ניתן לפתוח"** — הרץ שוב את הסקריפט; החתימה מבוצעת מחדש בסוף. אם צריך: `xattr -dr com.apple.quarantine ~/Applications/ChatGPT-RTL.app`.
- **קורס מיד בהפעלה** — כמעט תמיד בעיית חתימה. הרצה חוזרת של הסקריפט בונה נקי מאפס.
- **הטקסט לא מיושר** — ודא שאתה מסתכל על העותק ה‑RTL (לא המקורי). RTL מופיע רק על טקסט שמכיל אותיות עבריות/ערביות.
- **חזר ל‑LTR אחרי עדכון** — ה‑watcher אמור לתקן; להפעלה מיידית פשוט הרץ שוב את הסקריפט.
